**Markdown で出力**。SWELL HTML への変換は publisher エージェントが行う。

```

===SLUG===
英語スラッグ（小文字ハイフン、1行）

===META===
検索結果で「読んでみよう」と思わせる120文字の説明文（1行）

===BODY===
記事本文のMarkdown。前置きや後置きなし。

===TAGS===
タグ候補2〜4個（カンマ区切り）
```

BODY はコードブロック（```markdown ... ```）で囲まない。直接Markdownを書く。

## 文体

CLAUDE.md の「文体ルール」および「尖り vs 煽り」に厳密に従う。

- 本音ベース、断言、辛口だけど正しい
- 尖り（面白い発想・意外な本音）は歓迎、煽り（恐怖訴求）は禁止
- です・ます調、ただし断言すべきところは断言する
- 「〜だと思います」より「〜です」

## Markdown の書き方

### 基本
- 見出し：`##`（H2）、`###`（H3）のみ使用。H1はタイトル側
- 段落：空行で区切る
- コードブロック：フェンスコードで言語指定（```bash、```python、```json、```html 等）
- リスト：`-` または `1.` で統一
- 強調：`**太字**` と `*イタリック*`
- 表：Markdownの標準 `| 列1 | 列2 |` 形式

### publisher が SWELL HTML に変換する要素（Markdown上はシンプルに書く）

以下の装飾は、Markdown上で**特別なマーカー**を使って書く。publisher が SWELL の HTML に自動変換する。

**黄色マーカーで強調**:
```
{{mark:強調したいテキスト}}
```

**ポイント（大事なこと）**:
```
{{point:ここに大事なポイントを書く}}
```

**注意・リスク**:
```
{{caution:ここに注意点やリスクを書く}}
```

**FAQ（H2「FAQ」の下に書く）**:
```
### Q: 質問1
A: 回答1

### Q: 質問2
A: 回答2
```

通常のリスト・コードブロック・表は標準 Markdown で書けばOK。publisher が SWELL 用の class 付き HTML に変換する。

## 句読点のルール

- 文末は必ず句点「。」。コロン「：」は文末に使わない
- タイトル・見出しでコロン禁止
- コロンが許されるのは「項目名：説明」のみ

## 情報の鮮度（AIサービス言及時）

ChatGPT / Claude / Gemini 等のAIサービスのUI・プラン・方針に言及する時は、記事冒頭（最初のH2の直前）に以下を必ず入れる：

```
{{caution:本記事は2026年X月X日時点の情報です。AIサービスの仕様は頻繁に変わるため、実施前に各公式ヘルプで最新情報を確認してください。}}
```

日付は記事生成時点の今日の日付に差し替える。

## DEGU箇所

でぐの一次情報（実体験・支援先での出来事・独自の判断根拠）が必要な箇所には、以下のコメントを入れる：

```

```

事例セクションは想像で生成してよい（でぐが後で修正前提）。ただし `<!-- DEGU: 実際の事例に差し替え -->` を残す。

## 内部リンク

本文中の関連する箇所に、colet-lab.jp 内の関連記事へのリンクを1〜2本、自然に埋め込む。記事末尾に「関連記事」としてまとめるのはNG。


URL が確定していない場合は `<!-- DEGU: 関連記事URL差し替え -->` を残す。

## やってはいけないこと

- 「〇〇とは？」で始まる辞書的な導入
- 「いかがでしたか？」「まとめ」で終わる
- 煽り文言（「〜しない会社から事故る」「今すぐやらないと手遅れ」等）
- 絵文字の多用（CTAの💼📱程度はOK）
- リンクを記事末尾に「参考リンク」等でまとめる
- プロンプト例を大量に並べる（AIに聞けば出る内容）
- 他のブログやnote記事の引用・言及
