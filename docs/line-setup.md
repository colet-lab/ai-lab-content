# LINE公式 運用設計

## 位置付け

LINE公式を**「ラボの棚へのアクセス権」**として設計。最優先は登録者数増。

### 優先順位
**A 登録者数 > C 資料DL > B 法人相談 > D 回遊**

## 登録インセンティブ

**ラボの棚（資料図書館）へのアクセス**が唯一の登録インセンティブ。

- 登録すると今月のパスワードが届く
- パスワードで /resources/ 配下の全資料にアクセス可能
- 月1回パスワード変更 → 登録解除者は翌月からアクセス不可
- 資料は記事を書くたびに増える → 長く留まる理由

スタートキットを先に作るのではなく、記事作成の副産物として資料が貯まる。

## ステップ配信（登録後7日間・自動配信）

| タイミング | 内容 | 目的 |
|---|---|---|
| **Day 0（即時）** | ウェルカム + 今月の資料室パスワード + まず読む1本の資料リンク | 価値を即届ける・初動離脱防止 |
| **Day 1** | 自己紹介 + 「AIに詳しい必要はない」の哲学（note-guide.md の要約） + 資料室の使い方 | 人となりを伝える・資料室を触らせる |

| **Day 5** | 「AIで事故った話／ヒヤリハット」 | 共感 + 危機感（煽りなし） |
| **Day 7** | 相談ルート案内 + セミナー告知（あれば） | B 法人相談への橋渡し |

## 通常配信

### 頻度・曜日・時間
**週2本：火12:30 + 金18:30**

- 毎日配信はNG（ブロック率上昇）
- 火12:30 = ランチタイム開封率最大
- 金18:30 = 退勤時間、週末の読書候補
- 月水木日は避ける

### 内容配分
- **新記事案内 40%**：「今週の記事、ここが怖い／面白い」の1文コメント
- **Tips・読み物 40%**：LINE限定の「さっき試して面白かった」系短文
- **資料・セミナー告知 20%**：月次パスワード更新・新資料追加・イベント

### トーン
- ブログより少しフランク、ただし密度は落とさない
- 絵文字は1配信0〜2個
- 1配信300〜500字上限

## 月初のパスワード配信（テンプレ）

月初に以下を配信する：

```
【2026年X月】資料室パスワード

◆ 今月のパスワード: colet-lab-26XX

◆ 新規公開資料
- 資料名（v1.0）

◆ 更新資料
- 資料名（v1.2）
  → 何を変えたかの1文

資料室はこちら → colet-lab.jp/resources/
```

## タグ設計（標準機能内）

| タグ分類 | 内容 |
|---|---|
| **登録経路** | AI Lab記事 / YouTube / X / 資料請求 / 紹介 |
| **属性推定** | 経営者 / 情シス / エンジニア |
| **行動** | 資料DL済 / 問い合わせ済 / セミナー参加済 |

標準機能では細かい自動タグ付けは限定的。500人超えたら Lステップ等の高機能ツール導入を検討。

## 登録を増やす施策

1. **HP記事の末尾CTAに LINE登録導線**
2. **YouTube動画の説明文・エンディングで誘導**
3. **X/Threads/Instagram の固定ポストに LINE QR**
4. **名刺QR コードに LINE QR を入れる**
5. **既存取引先・フォロワーに初動で案内**
6. **セミナー参加者全員に登録案内**

## Lステップ等の高機能ツール

現時点は**不要**。登録者500人超えたら検討。

### 検討タイミング
- 登録者500人超
- 資料DLした人だけに別シナリオ配信したい
- 流入経路を細かく測定したい
